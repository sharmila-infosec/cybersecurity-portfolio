# Network Security — Wireshark vs tcpdump

## Project Description

In this project, I created a comparison diagram of two widely used network packet analysis tools — **Wireshark** and **tcpdump**. The diagram highlights their similarities, differences, and ideal use cases to demonstrate understanding of network traffic analysis tools used in SOC environments.

## Tools Compared

### Wireshark
- GUI-based network protocol analyzer
- Best for deep, visual packet inspection
- Easy to navigate and visualize traffic patterns
- Ideal for detailed forensic analysis

### tcpdump
- Command-line packet capture tool
- Lightweight and fast
- Best for quick captures in terminal environments
- Commonly used in Linux/Unix server environments

## Similarities

Both Wireshark and tcpdump:
- Capture network packets in real time
- Are open-source and freely available
- Support packet filtering to isolate specific traffic
- Are used for network troubleshooting and security investigations

## Skills Demonstrated

- Network Traffic Analysis
- Packet Capture and Inspection
- Wireshark
- tcpdump
- Network Protocol Understanding
- SOC Tool Comparison and Selection

## Key Takeaways

- Wireshark is preferred when visual analysis and detailed protocol inspection are needed
- tcpdump is preferred in command-line or resource-constrained environments
- Both tools are essential for SOC analysts performing network-level threat detection and incident investigation
- Understanding when to use each tool improves efficiency during security investigations
